# frp-config-reference

frp v0.65.0版本 配置参考

直接查看文件夹中的`.toml`文件，里面都有详细注释；`.toml`是frp目前使用的配置格式；文件开头的注释注明了这个文件配置到哪个设备上

需要说明的是：
* 对于homelab暴露服务到公网这一示例，采用的是tcp模式，优点是访问服务的设备不用运行frpc，缺点是任何人都可以访问服务
* 对于windows远程桌面这一示例，采用stcp模式，优点是服务对外部不可见，缺点是访问服务的设备也要运行frpc
* 这两个模式实际上不与使用场景绑定，可以根据需求权衡用哪个模式
* 对于linux，我个人不推荐使用远程桌面（毕竟windows是基于窗口的而linux是基于命令终端的），更推荐使用ssh；如需远程访问ssh，配置起来是一样的，把端口22代理出去即可（当然更推荐先把ssh端口改到高位，然后把对应高位端口代理出去）
